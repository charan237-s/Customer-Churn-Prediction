# Prestige — Customer Churn Prediction

An AI-powered dashboard that predicts whether a telecom customer is likely to
churn (cancel their service), built with Streamlit and scikit-learn.

## Project Structure

```
your-project/
├── app.py                        # Streamlit dashboard (Prediction / History / About)
├── main.py                       # Data cleaning, EDA, and model training script
├── requirements.txt              # Python dependencies
├── data/
│   ├── customer_churn.csv        # Raw dataset (you provide this)
│   ├── customer_churn_clean.csv  # Generated by main.py
│   └── prediction_history.csv    # Generated by app.py as predictions are made
├── models/
│   ├── churn_model.pkl           # Generated by main.py
│   └── scaler.pkl                # Generated by main.py
└── images/
    └── *.png                     # EDA charts generated by main.py
```

## 1. Setup

```bash
git clone <your-repo-url>
cd your-project
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

You'll also need `matplotlib` and `seaborn` if you plan to run `main.py`
(they're not required to run `app.py` alone):

```bash
pip install matplotlib seaborn
```

## 2. Get the dataset

Download the **Telco Customer Churn** dataset (7,043 rows, 19 features + target)
and place it at:

```
data/customer_churn.csv
```

## 3. Train the model

```bash
python main.py
```

This will:
- Load and clean the dataset (remove duplicates, fix `TotalCharges`, fill missing values)
- Save the cleaned dataset to `data/customer_churn_clean.csv`
- Generate EDA charts into `images/`
- Encode features, split train/test, scale, and train a `RandomForestClassifier`
- Print accuracy, classification report, and confusion matrix
- Save `models/churn_model.pkl` and `models/scaler.pkl`

**Run this before starting the app** — `app.py` will refuse to start if these
two files don't exist yet.

## 4. Run the app

```bash
streamlit run app.py
```

Open the URL shown in the terminal (usually `http://localhost:8501`).

### Pages
- **Prediction** — enter a customer's details and get a churn prediction with probability
- **History** — view, download, or clear all saved past predictions
- **About** — model details and how the app works

## 5. Deploy (Streamlit Community Cloud)

1. Push this project to a GitHub repo — **make sure `models/*.pkl` is committed**
2. Go to [share.streamlit.io](https://share.streamlit.io) and sign in with GitHub
3. Click **New app** → select your repo/branch → set main file to `app.py`
4. Click **Deploy**

Note: Streamlit Cloud's filesystem resets on redeploy/restart, so
`data/prediction_history.csv` won't persist long-term there — fine for a demo,
not for production data storage.

## Notes on the encoding

The categorical encodings in `app.py`'s `maps` dict must exactly match the
encodings used to train the model in `main.py`'s `encode_maps` dict. If you
retrain the model with different category mappings or column order, update
both files together, or predictions will be silently wrong.
