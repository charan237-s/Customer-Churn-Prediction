# Prestige — Customer Churn Prediction

**🔴 [Live Demo](https://customer-churn-prediction-pckr.streamlit.app/)**

An AI-powered dashboard that predicts whether a telecom customer is likely to
churn (cancel their service), built with Streamlit and scikit-learn.

## Project Structure

```
your-project/
├── .streamlit/
│   └── config.toml               # Dark theme config (prevents white flash on load)
├── .gitignore
├── LICENSE                       # MIT
├── app.py                        # Streamlit dashboard (Prediction / History / About)
├── main.py                       # Offline data cleaning, EDA & model training script
├── requirements.txt              # Python dependencies
├── data/
│   ├── customer_churn.csv        # Raw dataset (you provide this)
│   ├── customer_churn_clean.csv  # Generated by main.py
│   └── prediction_history.csv    # Generated by app.py as predictions are made
├── models/
│   ├── churn_model.pkl           # Generated by main.py
│   ├── scaler.pkl                # Generated by main.py
│   ├── metrics.json              # Generated by main.py — accuracy/precision/recall/F1
│   └── feature_importance.csv    # Generated by main.py — ranked feature importances
└── images/
    └── *.png                     # EDA charts generated by main.py
```

## Important: `main.py` is offline-only

`main.py` does the heavy lifting — cleaning, EDA, comparing models, training,
and saving artifacts. **It is never run as part of the deployed app.**
`app.py` never imports or executes it; it only ever loads the `.pkl`/`.json`
files `main.py` produces. Run `main.py` once locally (or re-run it whenever
you update the dataset) — deployment should only ever point at `app.py`.

## 1. Setup

```bash
git clone <your-repo-url>
cd your-project
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

`main.py` also needs `matplotlib` and `seaborn` for the EDA charts (not
required to run `app.py` alone):

```bash
pip install matplotlib seaborn
```

## 2. Get the dataset

Download the **Telco Customer Churn** dataset (7,043 rows, 19 features + target)
and place it at:

```
data/customer_churn.csv
```

## 3. Train the model

```bash
python main.py
```

This will:
- Load and clean the dataset (remove duplicates, fix `TotalCharges`, fill missing values)
- Save the cleaned dataset to `data/customer_churn_clean.csv`
- Generate EDA charts into `images/`
- Encode features and split into train/test sets
- **Compare three candidate models** (Logistic Regression, Random Forest,
  Gradient Boosting) via 5-fold cross-validation and automatically keep the
  best performer
- Print accuracy, precision, recall, F1, and a confusion matrix
- Save:
  - `models/churn_model.pkl` — the trained model
  - `models/scaler.pkl` — the fitted feature scaler
  - `models/metrics.json` — test-set performance + training metadata
  - `models/feature_importance.csv` — ranked feature importances

**Run this before starting the app** — `app.py` will refuse to start if
`churn_model.pkl` and `scaler.pkl` don't exist yet. `metrics.json` and
`feature_importance.csv` are optional — if missing, the app falls back to
placeholder stats on the dashboard and About page.

## 4. Run the app

```bash
streamlit run app.py
```

Open the URL shown in the terminal (usually `http://localhost:8501`).

### Pages & Features

**Prediction**
- **Single Customer** tab — enter one customer's details, get a churn
  prediction with probability and a Low / Medium / High risk badge
- **Batch Upload (CSV)** tab — upload a CSV of many customers at once and get
  predictions + probabilities for all of them, with a results download
- **Reset Form** button to clear all inputs back to defaults
- **Download This Result** — export a single prediction as its own CSV
- Model is loaded once and cached (`st.cache_resource`) for fast reruns

**History**
- Every single-customer prediction is saved automatically
- Filter by outcome (All / Churn / Stay) and sort by tenure, monthly charges,
  total charges, or most recent
- Churn-vs-stay ratio bar and summary metrics
- Download full history as CSV, or clear it

**About**
- Live model specs pulled from `models/metrics.json` (model name, accuracy,
  precision, recall, F1, dataset size, last trained date) — falls back to a
  note to run `main.py` if metrics aren't available yet
- Top predictive features chart, pulled from `models/feature_importance.csv`

## 5. Deploy (Streamlit Community Cloud)

1. Push this project to a GitHub repo — **make sure `models/*.pkl`,
   `models/metrics.json`, and `models/feature_importance.csv` are committed**
2. Go to [share.streamlit.io](https://share.streamlit.io) and sign in with GitHub
3. Click **New app** → select your repo/branch → set main file to `app.py`
4. Click **Deploy**

Note: Streamlit Cloud's filesystem resets on redeploy/restart, so
`data/prediction_history.csv` won't persist long-term there — fine for a demo,
not for production data storage.

## Notes on the encoding

The categorical encodings in `app.py`'s `COLUMN_ENCODINGS` dict must exactly
match the encodings used to train the model in `main.py`'s `ENCODE_MAPS` dict
— they're written to mirror each other intentionally. If you retrain the
model with different category mappings or column order, update both files
together, or predictions will be silently wrong. The batch-upload CSV must
also use these exact column names and category spellings (see the in-app
instructions on the Batch Upload tab).
